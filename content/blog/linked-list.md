關於 Linked List

是一種用「指標（reference）」串起來的資料結構，而不是用連續記憶體。

把這個「Linked List」實體化想像成「火車」，

每一節車廂都可以連結另一節車廂。

---

以 JavaScript 作為範例

```
[10, 20, 30]
```

在記憶體裡面是這樣：

```
[10][20][30]
 ↑   ↑   ↑
連續的一排
```

---

| 能力        | Array             | Linked List        |
| --------- | ----------------- | ------------------ |
| 取第 i 個    | O(1)              | O(n)               |
| 走訪        | 快（cache friendly） | 慢（pointer chasing） |
| 尾端 push   | O(1)              | O(1)               |
| 中間插入      | O(n)              | O(1)               |
| 中間刪除      | O(n)              | O(1)               |
| 記憶體       | 緊密                | 鬆散 + 指標            |
| CPU cache | 極快                | 很慢                 |

---

為了了解這兩樣不同之處

「實際上是硬體上寫入記憶的方式不同」
